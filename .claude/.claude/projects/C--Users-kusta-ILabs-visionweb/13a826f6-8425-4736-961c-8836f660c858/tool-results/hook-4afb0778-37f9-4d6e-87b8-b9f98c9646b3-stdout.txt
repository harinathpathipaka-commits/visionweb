Not just detect it вҖ” decide which eye to trust.\n\n### Resolution Hierarchy (which eye wins)\n\n| Conflict | Resolution Strategy | Rationale |\n|----------|-------------------|-----------|\n| **Vision vs DOM: visibility** | DOM wins on `is_visible`, Vision wins on `is_actually_visible` | DOM knows CSS computed style. Vision knows if element is visually obscured by overlay. |\n| **Vision vs DOM: element exists** | DOM wins | DOM is ground truth for element existence. Vision can hallucinate elements. |\n| **Vision vs DOM: element is blocked** | Vision wins | Only Vision can see overlays/blocking elements. DOM has no concept of z-order occlusion. |\n| **Vision vs DOM: text content** | DOM wins for exact text. Vision wins for \"what the user would read\" if text is truncated/overlapped by CSS. | DOM has exact textContent. Vision understands visual presentation. |\n| **DOM vs Diff: page changed** | Diff wins | Diff is the computed change. DOM is a point-in-time snapshot. If Diff says \"nothing changed\" but DOM reader says \"new content appeared\", trust Diff (it compared the trees). |\n| **Goal Verifier vs Error Detector: did action succeed?** | Goal Verifier wins on \"did goal advance?\" Error Detector wins on \"was there an error?\" | They measure different things. If Goal Verifier says \"goal didn't advance\" AND Error Detector says \"no errors\" вҶ’ action was successful but irrelevant (GOAL_DRIFT). |\n| **Vision vs Injection Detector: dangerous content?** | Injection Detector wins | Injection Detector pattern-matches instruction-override patterns. Vision can't detect obfuscated text. |\n\n### Resolution Algorithm\n\n```python\ndef resolve_contradictions(eye_reports: list[EyeReport]) -> SituationalSnapshot:\n    snapshot = SituationalSnapshot()\n\n    for conflict in detect_conflicts(eye_reports):\n        winner = RESOLUTION_TABLE[conflict.conflict_type]\n        snapshot.add(winner.report, confidence=winner.confidence)\n        snapshot.add_conflict_log(conflict, resolved_by=winner)\n\n        # If confidence is low (<0.7) after resolution, flag for human\n        if winner.confidence < 0.7:\n            snapshot.add_uncertainty_flag(conflict)\n\n    return snapshot\n```\n\n### Concrete Example\n\n```\nVision:  \"Search button is visible at (400, 300)\"         confidence: 0.95\nDOM:     \"button.search-btn: is_visible=false, disabled\"  confidence: 1.0\n\nConflict: visibility of element \"button.search-btn\"\nResolution: DOM wins on interaction state (the button IS disabled).\n            Vision correctly saw the rendered pixels but missed the\n            disabled attribute.
